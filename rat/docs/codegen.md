# codegen
Code generation turns the floating graph back into linear code in three stages: schedule the graph into blocks, lower to a target-independent machine IR, allocate registers. Only the lowering step is target-specific.

## schedule
[`CodeGen/Schedule.h`:](../include/CodeGen/Schedule.h) global code motion. Recovers a CFG from the control edges (one block per region / entry projection / if projection, terminated by a return, a two-way branch, or a goto), computes dominators with Lengauer-Tarjan and loop depth from natural loops, then places each floating node into a block: as early as its inputs allow, then sunk toward its uses into the shallowest-loop-depth block dominated by the early placement (hoisting out of loops where legal). Blocks come out in RPO with their phis and compute nodes in emit order. Used by both the x86 backend and the C emitter.

## machine ir
[`CodeGen/MachineFunction.h:`](../include/CodeGen/MachineFunction.h) a minimal, target-independent instruction form: blocks of `MachineInstr`s with defs, uses, clobbers, a register class, and backend-defined immediates. Operands are virtual registers, physical registers, immediates, frame slots, symbols, or block references. Calls are flagged so allocators apply clobbers and bound live ranges correctly.

## register allocation
Both allocators share [`CodeGen/RegAllocBase.h`](../include/CodeGen/RegAllocBase.h): per-block liveness over dense vreg bitsets, a linearized instruction order, copy-hint collection, and a common rewrite step that patches assignments in and inserts spills/reloads. They are fully backend-agnostic (register classes come from the target's `RegisterInfo`, and spill/reload/slot construction goes through `RegAllocHooks` callbacks), so neither allocator names a single target opcode.
- [**linear scan:**](../include/CodeGen/LinearScanRegAlloc.h) Live ranges are hole-aware segment lists: the gaps between segments are provably off every def-use path, so fixed-register pins inside a hole don't constrain the value and call clobbers inside a hole don't force a callee-saved register. Assignment scans ranges in start order per class. Copy hints bias the choice so coalescable moves become elided self-moves. Under pressure a value spills to a frame slot.
- [**graph coloring:**](../include/CodeGen/GraphColorRegAlloc.h): Build the interference graph from liveness, simplify by repeatedly removing nodes whose degree is below the color count. When stalled, optimistically push the cheapest node (`spill cost = uses / span / degree`) and let the select phase decide. Selection pops the stack in reverse, coloring each node while respecting interference and precolored constraints, uncolorable nodes spill to frame slots. Nodes whose adjacency reaches 128 are marked must-spill up front and never enter the coloring stack.
